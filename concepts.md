## 1. Agent 的定义与分类

### 1.1 什么是 Agent?

**简述**
Agent（智能体）是一个能够：
- 👁️ 感知环境（Perceive）
- 🧠 进行推理（Reason）
- 🎯 制定计划（Plan）
- 🚀 执行动作（Act）
- 📚 从经验学习（Learn）

的自主程序。

**正式定义（Russell & Norvig）**
Agent = 体系结构（Architecture）+ 程序（Program）

其中：
- 体系结构：物理设备和软件基础设施
- 程序：决策与行为的算法

### 1.2 Agent 的关键特性

| 特性 | 说明 | 示例 |
|------|------|------|
| **自主性** | 独立做出决策，无需人工干预 | Agent 自主选择工具调用 |
| **反应性** | 对环境变化快速响应 | 实时检测用户输入并回应 |
| **主动性** | 积极主动达成目标 | 主动制定计划实现目标 |
| **社交性** | 与其他 Agent 协作通信 | 多 Agent 系统中的协作 |
| **学习性** | 从交互中改进行为 | 持续优化决策策略 |

### 1.3 Agent 的分类

**按设计范式**

```
┌─ 反应式 Agent（Reactive）
│  ├─ 定义：直接对感知刺激做出反应
│  ├─ 特点：无内部状态、快速响应、规则简单
│  └─ 例子：爬虫、传感器控制程序
│
├─ 延迟式 Agent（Deliberative）
│  ├─ 定义：建立环境模型，规划后执行
│  ├─ 特点：有内部状态、计划性强、适应复杂环境
│  └─ 例子：路径规划算法、任务调度系统
│
└─ 混合式 Agent（Hybrid）
   ├─ 定义：结合反应和延迟的优点
   ├─ 特点：既快速响应又能规划
   └─ 例子：现代 LLM Agent、自动驾驶
```

**按所处层级**

1. **单体 Agent**：独立工作
2. **多 Agent 系统**：多个 Agent 协作
3. **分层 Agent 系统**：主-从结构

**按智能程度（当前阶段）**

1. **LLM 基础 Agent**：基于 LLM 的直接推理
2. **工具增强 Agent**：具有工具调用能力
3. **记忆增强 Agent**：有短期和长期记忆
4. **自主规划 Agent**：能递归分解任务
5. **多 Agent 系统**：支持协作与竞争

### 2.2 关键子系统详解

**1. 记忆系统（Memory System）**

```
短期记忆（Short-term Memory）
├─ 当前对话历史
├─ 最近执行的动作
├─ 当前上下文信息
└─ 容量有限（Token 限制）

长期记忆（Long-term Memory）
├─ 知识库（文档、数据）
├─ 学习经验
├─ 任务历史
└─ 持久存储（数据库）

工作记忆（Working Memory）
├─ 中间推理结果
├─ 中间计算值
├─ 当前目标状态
└─ 临时存储区
```

**2. 推理引擎（Reasoning Engine）**

当代 Agent 的推理核心：
- 🧠 LLM（Large Language Model）作为推理骨干
- 📖 知识库提供背景知识
- 🔗 链式推理（Chain-of-Thought）进行多步推导
- 🎯 约束求解进行最优决策

**3. 决策模块（Decision Module）**

```
决策过程：
    1. 评估状态：当前环境与目标差距
    2. 生成选项：LLM 生成可能的动作
    3. 评估选项：计算每个动作的价值
    4. 选择最优：根据评分选择最佳动作
    5. 执行动作：调用工具或 API 执行
    6. 获取反馈：观察结果并更新状态
```

**4. 工具系统（Tool System）**

Agent 的能力扩展机制：
- API 接口：Web 服务、数据库
- 代码执行：Python、SQL 解释器
- 搜索工具：Web、本地文档搜索
- 计算工具：数学、统计计算
- 生成工具：文本、代码、图像生成

## 3. LLM 与 Agent 的关系

### 3.1 LLM 在 Agent 中的角色

```
┌──────────────┐
│   用户输入   │
└───────┬──────┘
        │
        ↓
┌──────────────────────────────────┐
│  LLM：理解、推理、规划、决策       │
│  核心问题："现在该做什么？"       │
├──────────────────────────────────┤
│  • 理解用户意图                  │
│  • 分析当前状态                  │
│  • 制定执行计划                  │
│  • 选择最佳行动                  │
│  • 组织输出结果                  │
└──────────────┬───────────────────┘
               │
        ┌──────┴──────┐
        │             │
        ↓             ↓
    ┌────┐        ┌────┐
    │工具│        │回复│
    └────┘        └────┘
```

### 3.2 LLM 的关键能力

**1. 理解能力**
- 理解自然语言
- 识别隐含意图
- 提取关键信息

**2. 推理能力**
- 逻辑推导
- 常识推理
- 多步推理

**3. 规划能力**
- 任务分解
- 步骤排序
- 资源分配

**4. 生成能力**
- 文本生成
- 代码生成
- 结构化输出

### 3.3 LLM 的局限性与补充

**LLM 的局限**
- ❌ 知识可能过时
- ❌ 推理容易出错
- ❌ 无法直接调用外部系统
- ❌ 上下文窗口有限
- ❌ 计算能力有限

**Agent 如何补充**

| LLM 的限制 | Agent 的解决方案 |
|-----------|----------------|
| 知识过时 | 集成 RAG 检索最新信息 |
| 推理出错 | 工具验证、反馈调整 |
| 无系统调用 | 工具调用机制 |
| 上下文限制 | 记忆管理、信息筛选 |
| 计算能力弱 | 调用计算工具 |

## 4. 关键技术详解

### 4.1 工具调用（Tool/Function Calling）

**原理**

```
┌────────────────────────────────────────┐
│  1. 定义工具                           │
│  - 工具名称、描述、参数定义            │
└────────────────────────────────────────┘
         ↓
┌────────────────────────────────────────┐
│  2. 发送给 LLM                         │
│  - 在 Prompt 中包含工具定义            │
│  - 告诉 LLM "你可以调用这些工具"      │
└────────────────────────────────────────┘
         ↓
┌────────────────────────────────────────┐
│  3. LLM 决策                           │
│  - 理解用户请求                        │
│  - 判断是否需要调用工具                │
│  - 生成工具调用指令                    │
└────────────────────────────────────────┘
         ↓
┌────────────────────────────────────────┐
│  4. 执行工具                           │
│  - 解析工具调用                        │
│  - 执行相应操作                        │
│  - 获取执行结果                        │
└────────────────────────────────────────┘
         ↓
┌────────────────────────────────────────┐
│  5. 反馈给 LLM                         │
│  - 将结果发送回 LLM                    │
│  - LLM 继续推理或返回最终结果          │
└────────────────────────────────────────┘
```

**实现示例（伪代码）**

```python
# 1. 定义工具
tools = [
    {
        "name": "calculator",
        "description": "执行数学计算",
        "parameters": {
            "expression": "数学表达式"
        }
    }
]

# 2. 用户请求
user_query = "3加5等于多少？"

# 3. 调用 LLM 并附加工具定义
response = llm.complete(
    messages=[
        {"role": "user", "content": user_query}
    ],
    tools=tools
)

# 4. 检查是否需要调用工具
if response.tool_call:
    tool_name = response.tool_call["name"]
    params = response.tool_call["parameters"]
    
    # 5. 执行工具
    result = execute_tool(tool_name, params)
    
    # 6. 反馈给 LLM
    response = llm.complete(
        messages=[...],
        tool_results=[result]
    )
```

### 4.2 检索增强生成（RAG）

**为什么需要 RAG？**

问题：
- LLM 的训练数据有截止日期
- 需要引入最新信息或私有知识
- 减少幻觉（Hallucination）

解决方案：检索增强生成

**RAG 流程**

```
用户查询
  ↓
[1] 查询向量化
  ↓
[2] 向量相似度检索 → 获取相关文档片段
  ↓
[3] 组织 Prompt（包含检索结果）
  ↓
[4] LLM 生成答案
  ↓
最终答案（基于事实，减少幻觉）
```

**关键组件**

- **文档处理**：分块、清洗、格式化
- **向量化**：使用 Embedding 模型转换为向量
- **向量存储**：数据库（Pinecone、Weaviate）
- **检索算法**：余弦相似度、混合检索
- **融合策略**：将检索结果与 LLM 输出整合

### 4.3 链式思考（Chain-of-Thought, CoT）

**概念**

让 LLM 展示其推理过程，而不是直接给出答案。

**效果对比**

```
直接提问：
Q: 小明有 5 个苹果，又买了 3 个。他现在有多少个苹果？
A: 8

CoT 提问：
Q: 小明有 5 个苹果，又买了 3 个。他现在有多少个苹果？
    请一步步思考。
A: 第一步：小明原有 5 个苹果
  第二步：又买了 3 个
  第三步：总数 = 5 + 3 = 8
  答案：8 个苹果
```

**优势**
- 提高复杂推理的准确度
- 便于理解 LLM 的思考过程
- 便于发现和纠正推理错误

### 4.4 记忆管理

**短期记忆与上下文**

```
当前对话：
用户：你好，我是小明
Assistant：你好小明，很高兴认识你
用户：我的名字是什么？
Assistant：你是小明
↑ 记忆自动在对话历史中
```

**长期记忆的实现**

```
对话过程中关键信息提取
  ↓
存入向量数据库
  ↓
后续查询时自动检索
  ↓
融入当前对话上下文
```

**记忆类型**

1. **即时记忆**：当前 Token 窗口内
2. **短期记忆**：对话历史（最近 N 条）
3. **长期记忆**：知识库、用户信息
4. **程序记忆**：代码、规则、习惯

## 5. Agent 工作流程

### 5.1 完整的 Agent 执行循环

```
[START]
  ↓
┌─────────────────────────────────┐
│ 1. 接收输入                      │
│    - 用户问题或任务指令          │
│    - 环境观察（如适用）          │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 2. 上下文准备                    │
│    - 加载相关记忆                │
│    - 检索相关文档（RAG）         │
│    - 组织系统提示                │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 3. LLM 推理                      │
│    - 理解任务                    │
│    - 分析当前状态                │
│    - 制定执行计划                │
│    - 判断是否需要工具            │
└─────────────────────────────────┘
  ↓
┌──────────────────┬──────────────────┐
│ 4a. 工具调用     │ 4b. 直接回复     │
├──────────────────┼──────────────────┤
│ • 调用工具       │ • 生成最终答案   │
│ • 执行操作       │ • 返回结果       │
│ • 获取结果       │                  │
│ • 反馈给 LLM    │                  │
│ • 继续推理       │                  │
└──────────────────┴──────────────────┘
  ↓
┌─────────────────────────────────┐
│ 5. 输出与学习                    │
│    - 返回最终结果                │
│    - 保存执行历史                │
│    - 提取关键经验                │
└─────────────────────────────────┘
  ↓
[END]
```

### 5.2 多轮交互与迭代

```
用户：帮我分析销售数据
   ↓
Agent：需要哪些数据？时间范围是什么？
用户：2024 年全年，按地区分析
   ↓
Agent：[调用数据查询工具] → 获取数据 → 分析 → 可视化
   ↓
Agent：生成报告（包含图表、建议）
用户：能重点分析东部地区吗？
   ↓
Agent：[利用已有数据进行细粒度分析]
   ↓
Agent：返回东部地区深度分析报告
```

## 6. 常见设计模式

### 6.1 单 Agent 模式

**Simple Agent**
```
输入 → LLM → 输出
```
适用：简单任务

**Tool-using Agent**
```
输入 → LLM → 判断是否需要工具
        ↓
        工具调用 → 结果 → LLM → 输出
```
适用：需要实时信息或计算的任务

**ReAct Agent（Reasoning + Acting）**
```
输入 → LLM 思考 → 决定动作 → 执行 → 观察结果
       ↑←←←←←←←←←←←←←←←←←←←←↓
   (反复循环直到完成)
```
适用：复杂多步骤任务

### 6.2 多 Agent 模式

**顺序执行模式**
```
Agent A 完成 → Agent B 接手 → Agent C 最终处理 → 结果
```
适用：流程化任务

**并行执行模式**
```
Agent A ┐
Agent B ├→ 聚合 → 最终结果
Agent C ┘
```
适用：多维度分析

**层级模式**
```
    主 Agent
    /  |  \
  子 Agent 们
```
适用：复杂组织结构任务

**竞争模式**
```
Agent A 生成方案1
Agent B 生成方案2
主Agent 评估并选择最优方案
```
适用：创意生成、决策优化

### 6.3 特殊模式

**Self-Critique 模式**
```
Agent 生成答案
  ↓
Agent 自我评估
  ↓
发现问题？
  ├─ 是 → 重新思考 → 改进答案
  └─ 否 → 返回答案
```

**Plan-and-Execute 模式**
```
Agent 制定详细计划
  ↓
逐步执行计划中的每个步骤
  ↓
根据实际情况灵活调整
  ↓
完成任务
```

## 总结

**Agent 的核心公式**

$$Agent = LLM + Memory + Tools + Loop$$

- **LLM**：推理与决策引擎
- **Memory**：信息存储与检索
- **Tools**：能力扩展与环境交互
- **Loop**：反复迭代与持续优化

